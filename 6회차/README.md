# 31장 RegExp

1. 정규 표현식이란 무엇이며, 자바스크립트에서 어떤 기능을 제공하나요?
    - 종훈리: 정규 표현식은 일정한 패턴을 가진 문자열의 집합을 표현하기 위해 사용하는 언어입니다. 문자열을 대상으로 패턴 매칭 기능을 제공합니다.
2. 정규 표현식을 생성하는 두 가지 방법과 차이점은 무엇인가요?
    - 채온: 정규 표현식 리터럴 방식 `/pattern/flags` / RegExp 생성자 함수 방식 `new RegExp(pattern[, flags])`
3. RegExp.prototype.test와 exec의 차이점은 무엇인가요?
    - 소연: test - 불리언 값으로 반환 / exec - 배열로 반환
4. 정규 표현식에서 사용하는 주요 플래그는 어떤 것이 있고 각각 어떤 기능을 하나요?
    - 히범: 3가지! i → 이그노어! 대소문자 구별않고 검색! g → 글로발! 어… 문자열 전체에서 해당 하는 패턴 다 찾아라~ m → 멀티라인 문자열 행 바뀌어도 계속 패턴 검색 하시오!
5. 정규 표현식에서 `.`과 `^`, `$`, `{m,n}`, `[...]`, `[^...]` 등의 의미는 각각 무엇인가요?
    - 종훈리: .은 임의의 문자 한 개
        
        ^는 배열 안, 밖에 따라 다르다. […] 안의 ^은 NOT의 의미, 밖의 ^는 문자열의 시작
        
        $는 문자열의 마지막
        
        {m, n}는 앞선 패턴이 최소 m번, 최대 n번 반복되는 문자열

# 32장 String

1. String 생성자 함수는 new 연산자 유무에 따라 어떤 차이를 보이나요?
    - 채온: new 사용: String 래퍼 객체 생성 / new x: 문자열을 반환
2. 문자열은 불변(immutable)인데, String의 메서드들은 어떻게 동작하나요?
    - 소연: String 객체의 메서드는 언제나 새로운 문자열을 반환.
    
    즉 문자열은 변경 불가능한 원시 값이기 때문에 String 래퍼 객체도 읽기 전용 객체로 제공됨.ㅋㅋ
    
3. `indexOf`와 `search` 메서드의 차이점은 무엇인가요?
    - 히범: 둘다 문자열 검색하는 메서드인데 인덱스오브는 인수로 문자열을 전달해서 찾고 서치는 정규표현식을 전달해서 찾음 둘다 첫번째로 찾은 녀석의 인덱스를 반환하고 못 찾으면 -1 반환함!
4. `slice`와 `substring`의 차이점은 무엇인가요?
    - 종훈리: 둘 다 문자열 검색하는 메서드이고, 둘의 동작 방식 또한 같다. 차이점이 있다면 slice는 subString 메서드와 달리 음수인 인수를 전달할 수 있다.
5. 대소문자 변경, 공백 제거, 문자열 반복 등에 사용하는 String 메서드는 어떤 것이 있나요?
    - 채온: 대소문자 변경 toUpperCase, toLowerCase /공백 제거 trim / 문자열 반복 repeat
6. 정규 표현식을 활용한 문자열 치환에는 어떤 메서드를 사용하며, 그 사용법은 어떻게 되나요?
    - 소연: ‘Hello so yeon’.replace(정규표현식 or 문자열, 치환 문자열)

# 35장 스프레드 문법

1. 스프레드 문법이란 무엇이며, 어떤 경우에 사용하나요?
    - 히범: … 로 표현하고 하나로 뭉쳐져있는 여러 값들을 풀어서 개별적인 값들의 목록으로 만들어준다!
    3가지가 있지!
        1. 함수 호출문의 인수 목록
        2. 배열 리터럴의 요소 목록
        3. 객체 리터럴의 프로퍼티 목록
    
    중요한건! 이터러블한 녀석들한테만 사용가능하다!는 것이지!
    
2. 스프레드 문법과 Rest 파라미터의 차이점은 무엇인가요?
    - 종훈리: 둘 다 … 형태로 나타나는 공통점. 차이점은 Rest 파라미터는 매개변수 앞에 …을 붙여서 인수를 전달하는 것. 스프레드는 배열과 같이 여러 개의 값이 하나로 뭉쳐 있는 이터러블을 펼쳐서 개별적인 값의 목록을 만드는 것.
    
    ```jsx
    function foo(...rest) { // Rest 파라미터
        console.log(rest);
    }
    foo(...[1, 2, 3]); // 스프레드
    ```
    
3. 함수 호출 시 스프레드 문법을 사용하면 어떤 장점이 있나요?
    - 채온: 배열에 담긴 값을 개별 인자로 전달할 수 있어서 `apply()` 없이 `…`으로만 사용할 수 있어서 더 간결하고 가독성이 좋다
4. 배열 리터럴 내부에서 스프레드 문법을 사용할 때 어떤 점이 편리한가요?
    - 소연: 간결하고 가독성이 좋은데, 배열을 결합/추가/삭제/복사할 때 메서드를 사용하지 않고 스프레드 문법으로만 할 수 있습니다
5. 기존의 배열 결합(concat)이나 삽입(splice) 기능을 스프레드 문법으로 어떻게 대체할 수 있나요?
    - 히범:
    
    ```jsx
    const arr = [...[1,2], ...[3,4]];
    
    console.log(arr);// [1,2,3,4]
    
    const arr1 = [1,4];
    const arr2 = [2,3];
     
    arr1.splice(1,0,...arr2); // [1,2,3,4]
    ```
    
6. 스프레드 문법을 객체 리터럴 내부에서 사용할 수 있나요? 가능하다면 어떻게 동작하나요?
    - 종훈리: 네. Rest 프로퍼티와 함께 스프레트 프로퍼티를 사용하면 객체 리터럴의 프로퍼티 목록에서도 스프레드 문법을 사용할 수 있습니다.
7. 스프레드 문법의 사용 대상은 무엇인가요? 객체도 사용할 수 있는 이유는 뭔가요?
    - 채온: 스프레드 문법은 이터러블 객체에 사용되는데, 스프레드 프로퍼티 제안은 일반 객체를 대상으로 사용 허용
